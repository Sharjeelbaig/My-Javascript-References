# My Tech Stack
This document outlines the technology stack used in my projects

- **Type Safety**: [TypeScript](https://www.typescriptlang.org/) - A strongly typed programming language that builds on JavaScript, giving you better tooling at any scale.
- **Languages**: 
  - [JavaScript (ES6+)](https://developer.mozilla.org/en-US/docs/Web/JavaScript) - A high-level, often just-in-time compiled language that conforms to the ECMAScript specification.
  - [Python](https://www.python.org/) - A high-level, interpreted programming language known for its readability and versatility.
- **BE Frameworks**: [Hono](https://hono.dev/) - A small, fast, and simple web framework for building serverless applications.
[FastAPI](https://fastapi.tiangolo.com/) - A modern, fast (high-performance) web framework for building APIs with Python 3.7+ based on standard Python type hints.
[Flask](https://flask.palletsprojects.com/en/2.3.x/) - A lightweight WSGI web application framework in Python.
- **Database Tech**: [Neon](https://neon.tech/) - A serverless Postgres database for modern applications.
- **Firebase Admin SDK**: [Firebase Admin](https://firebase.google.com/docs/admin/setup) - Used for server-side Firebase functionalities such as validating ID tokens.
- **Authentication**: [Firebase Authentication](https://firebase.google.com/products/auth) - Provides backend services for easy use of authentication in applications.
- **Storage**: [Firebase Storage](https://firebase.google.com/products/storage) - Used for storing and serving user-generated content.
- **Generative AI**: [Ollama](https://ollama.com/) - A platform for running large language models locally.
[Ollama Cloud](https://ollama.com/cloud) - An inference API for accessing LLMs in the cloud.
[LangChain](https://langchain.com/) - A framework for developing applications powered by language models.
- **Frontend Framework**: [React](https://react.dev/) - A popular JavaScript library for building user interfaces.
- **Cloudflare Pages**: [Cloudflare Pages](https://pages.cloudflare.com/) - Will be used for deploying frontend
- **Vercel**: [Vercel](https://vercel.com/) - Will be used for deploying backend
